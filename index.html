MinIR API

MinIRで利用できるWebAPIの説明です。

事前準備

MinIRへのデータ書き込みにはPublicKeyが必要です。 以下の手順で取得できます。

  1. MinIRにログインします。
  2. GENERATE API KEY ボタンを押してKeyを作成します。

※注意

  • GENERATE API KEYを押すたびにキーが再作成されます。表示されているキーのみ有効です。

GET SongScores

曲のスコアリストを取得します。

URL

https://minir.azurewebsites.net/api/getSongScores

HTTP メソッド

GET

クエリパラメータ

なし

ヘッダー

  • songhash (必須)： 譜面のsha256
  • lnmode （オプション）：lnmode(0 or Nothing:LN, 1:CN, 2:HCN)

※LNMODE拡張定義があるBMSはlnmode=0でしか取得できません。

BODY

なし

PUT SongScore

曲のスコアをMnIRに登録します。

URL

https://minir.azurewebsites.net/api/putSongScore

HTTP メソッド

PUT

クエリパラメータ

なし

ヘッダー

  • publickey (必須)： GENERATE API KEYで取得したKey
  • dryrun （オプション）：trueの場合、MinIRに書き込みを行わず、書き込み結果がどのようになるかがレスポンスとして返却されます。

※LNMODE拡張定義があるBMSはlnmode=0でしか取得できません。

BODY

曲スコアデータJSON。各項目は必須です。

{
    "songhash": "SONGHASH",
    "lnmode": 0,
    "score": 1234,
    "epg": 600,
    "lpg": 0,
    "egr": 33,
    "lgr": 1,
    "combo": 550,
    "clear": 3,
    "notes": 1555
}
  • songhash : 譜面のsha256
  • lnmode: LNプレイモード。(0:LN, 1:CN, 2:HCN) LNMODE拡張定義があるBMSはlnmode=0を指定してください。
  • score:EX SCORE
  • epg: early perfect great
  • lpg: late perfect great
  • egr: early great
  • lgr: late great
  • combo:max combo
  • clear
    • 0 : NoPlay
    • 1 : Failed
    • 2 : AssistEasy
    • 3 : LightAssistEasy
    • 4 : Easy
    • 5 : Normal
    • 6 : Hard
    • 7 : ExHard
    • 8 : FullCombo
    • 9 : Perfect
    • 10 : Max
  • notes : 譜面ノート数（※RANDOM譜面の場合、ノート数が変動する）

レスポンス例

Q&A

スコアが取得できない・送信できないんだけど？

  • クエリが間違っている
  • 高頻度でのリクエスト要求をしている

どちらかとは思いますが、単純にバグの可能性もあります。
どうしても解決しない場合はお問い合わせください。

高頻度でのリクエストは財布が死ぬからカンベンな！



どう考えても不正スコアが送信される作りだよね？

はい、スコア送信に関しては特にチェックを行っていません。
ただし、このAPIを利用して送信されたスコアはNo Validation スコアとして登録されます。
IRページでもNo Validationスコアは表示上区別されます。

レコメンドの算出等、他者比較が必要の場合はNo Validationスコアを集計対象から外すことが必要になってきます。

また、スコアの消去は実装してないので、変なスコア送ったとしても、No Validationスコアがずっと残るだけです…（恥ずかしい…恥ずかしくない？）
